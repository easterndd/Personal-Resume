# Product Requirements

> 状态：Active  
> 产品阶段：本地 MVP，面向服务器上线演进  
> 最后更新：2026-06-08

## 1. Product Statement

AI 简历工坊帮助用户创建、导入、修改和优化简历，并通过精选模板导出 PDF、DOCX、TXT 和 JSON。当前以本地 Web 验证核心体验，后续上线服务器并提供微信小程序客户端。

产品本身是简历生成平台。闲鱼等渠道只用于获客，不进入产品页面、数据模型或交互设计。

## 2. Target Users

### Current

- 产品开发者和本地运营者。
- 需要制作或修改简历的求职者。
- 需要为不同岗位生成多个简历版本的用户。

### Future

- Web 注册用户。
- 微信小程序移动用户。
- 在获得授权后协助客户交付简历的运营人员。

## 3. User Problems

- 不知道怎样组织和量化经历。
- 已有 PDF/DOCX 简历难以结构化修改。
- 模板选择多但缺少岗位适配建议。
- AI 容易捏造内容或输出不可直接使用的文本。
- PDF/DOCX 导出可能出现分页、字体和格式问题。
- 同一份经历需要针对多个 JD 生成不同版本。

## 4. Product Goals

- 跑通“创建/导入 -> 编辑 -> AI 优化 -> 预览 -> 导出”闭环。
- 支持多份简历和岗位版本管理。
- 使用结构化 ResumeData 保证多端和多格式复用。
- 提供可控、可比较、可撤销的 AI 建议。
- 提供少而精、有标签、有适用岗位说明的模板。
- 本地阶段保护简历隐私，线上阶段提供明确数据归属和删除能力。
- 为服务器部署和小程序复用保留稳定 API 边界。

## 5. Non-Goals

本地 MVP 不做：

- 公网注册、支付、会员和短信。
- 多租户团队协作。
- 模板投稿、社区、评论和积分。
- 完全自由的积木式设计器。
- 招聘网站抓取或自动投递。
- 独立的小程序业务逻辑。
- 微服务、Kubernetes 或复杂任务平台。

## 6. Core Journeys

### Create

1. 用户新建简历。
2. 填写基本信息、求职目标、经历、项目、教育和技能。
3. 实时预览模板。
4. 保存简历。
5. 导出 PDF 或其他格式。

### Import

1. 用户选择 PDF、DOCX、TXT 或 JSON。
2. 系统校验文件并提取内容。
3. 系统生成结构化草稿和解析提示。
4. 用户确认、修正后保存。

### AI Optimize

1. 用户选择目标岗位或输入 JD。
2. 系统诊断完整度、关键词和表达。
3. 用户选择单条、板块或全文建议。
4. 系统展示原文、建议、理由和待确认事实。
5. 用户明确应用后写回。

### Template and Export

1. 系统按行业、岗位和内容密度推荐模板。
2. 用户预览分页效果。
3. 系统执行导出前检查。
4. 用户下载文件并可查看导出记录。

## 7. MVP Scope

### P0

- 简历 CRUD 和复制。
- 结构化编辑、排序、实时预览。
- 3 套模板：modern、classic、compact。
- AI 单条改写、全文诊断、JD 匹配。
- PDF 与 JSON 导出。
- PDF、DOCX、TXT、JSON 导入的基础路径。
- API Key 本地配置入口。
- 本地数据与文件清理说明。

### P1

- DOCX 导出质量完善。
- 导出前分页、缺失字段、ATS 友好检查。
- 模板标签和推荐。
- AI 建议 diff、撤销和历史。
- 统一设置 API 与模型连接测试。
- 自动测试和 CI。

### P2

- 用户登录与简历归属。
- PostgreSQL、对象存储和后台任务。
- 云端删除、保留周期和审计。
- 微信小程序。
- 用量、套餐与支付。

## 8. Acceptance Criteria

- 用户能在全新环境按 Runbook 启动前后端。
- 用户能创建、保存、重新打开和复制简历。
- 导入失败不会覆盖已有简历，错误信息可理解。
- AI 请求失败不会丢失编辑内容。
- AI 建议不会自动写回，必须由用户确认。
- PDF 在 A4 下可阅读，中文字体正常，关键内容不被截断。
- JSON 导出可重新导入并保留核心字段。
- API Key、真实简历、上传文件和导出文件不会进入 Git。
- 移动视口可完成主要浏览和简单编辑；复杂排版仍以 Web 为主。

## 9. Success Metrics

本地验证期：

- 核心流程成功完成率。
- 一份简历从创建/导入到 PDF 导出的耗时。
- AI 建议采纳率和人工修改率。
- PDF/DOCX 导出失败率。
- 导入后需要人工修正的字段比例。
- 用户能否为一个 JD 生成可用版本。

上线后再增加：

- 注册到首份导出转化率。
- 次日/七日留存。
- 模型调用成本和单次导出成本。
- 数据删除请求完成时长。

## 10. Open Decisions

- 首次上线的登录方式。
- PostgreSQL 托管方案和对象存储供应商。
- API Key 由平台承担还是允许用户自带。
- 小程序采用原生、Taro 还是 uni-app。
- 计费与免费额度。
- 简历数据和原始文件的默认保留周期。
