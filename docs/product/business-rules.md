# Business Rules

> 状态：Active  
> 最后更新：2026-06-08

## Resume

- 一份简历有唯一 ID、标题、状态、模板、目标岗位、目标行业和结构化内容。
- 复制简历生成新 ID，状态重置为 `draft`，原简历不受影响。
- 简历状态只表示工作流阶段，不代表导出文件状态。
- 删除简历前必须二次确认；线上版还要校验资源归属。
- 保存失败时保留客户端编辑内容，并明确显示失败。
- 时间、空值和数组字段必须使用统一格式，禁止页面自行发明结构。

## AI

- AI 只能改写用户提供的事实，不得创造公司、岗位、奖项、技能、数字或经历。
- 缺少量化数据时只能提出问题或使用占位建议，不能编造数字。
- AI 输出必须与原文分离，默认不自动覆盖。
- 建议至少包含：建议文本、修改理由、风险或待确认信息。
- JD 匹配分数是辅助指标，不承诺面试或录用结果。
- 模型失败、限流或超时时，不得损坏简历数据。
- 服务器版 API Key 只存在服务端 Secrets；不得返回给 Web 或小程序。
- 发送给第三方模型前，应提示数据将离开本地环境，并尽量减少敏感字段。

## Import

- 允许格式：PDF、DOCX、TXT、JSON。
- 服务端必须验证扩展名、MIME、文件大小和解析结果，不能只相信文件名。
- 导入先生成草稿，用户确认后才保存为正式简历。
- 解析失败不能覆盖当前简历。
- JSON 导入必须经过 Schema 校验。
- 原始上传文件必须有生命周期；本地版提供清理方式，线上版提供自动过期和用户删除。

## Templates

- 模板必须基于同一 ResumeData，不得为单个模板发明不可迁移的数据字段。
- 模板差异主要由布局、字体、颜色、间距和显示策略构成。
- 每个模板必须有岗位、行业、风格、栏数、语言和 ATS 标签。
- 模板必须通过中文字体、A4 分页、空字段、长文本和多经历样例验证。
- 竞品模板只能用于研究，不能直接复制受版权保护的视觉或文件。
- MVP 不提供任意坐标和自由画布。

## Export

- PDF 面向投递和分享；DOCX 面向二次编辑；JSON 面向备份和迁移；TXT 面向纯文本使用。
- 文件名必须清理非法字符，且不能允许路径穿越。
- 导出失败不写入成功记录。
- 导出记录应包含简历 ID、格式、文件路径/对象键、创建时间和最终状态。
- 线上版下载地址应短期有效，不暴露服务器真实路径。
- 删除简历时必须定义导出文件是否一并删除。

## Privacy

- 默认不提交、不上传真实用户简历到源码仓库、Issue、日志或公共 AI 对话。
- 日志不得包含完整简历、API Key、Authorization Header 或原始文件内容。
- 线上数据必须按用户隔离，所有读写都执行对象级授权。
- 用户可导出自己的数据并删除简历、上传文件和导出文件。
- 备份中的删除保留策略必须在隐私政策中说明。
- 故障排查样例必须脱敏。

## Jobs

- 当前岗位推荐数据是 mock，不表示实时招聘岗位。
- 上线真实岗位能力前必须明确数据来源、更新时间和使用许可。
- 推荐结果只做辅助，不宣称与招聘方存在合作。

## Error Handling

- 用户错误返回可理解、可操作的信息。
- 内部堆栈、供应商响应和文件系统路径不得直接返回给线上客户端。
- 外部服务设置连接和读取超时。
- 只对幂等且可安全重试的请求执行有限重试。
- 错误发生后保留用户可恢复的编辑状态。
