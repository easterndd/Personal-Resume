# 闲鱼简历副业 — 协作落地计划书

> 两人团队：**小伙伴**负责闲鱼运营与客户对接 · **你**负责本地工具开发
> 模式：闲鱼获客 → 获取用户信息 → 你本地用工具生成简历 → 闲鱼交付

---

## 一、项目定位与分工

```
闲鱼客户 → 小伙伴沟通获取信息 → 整理给开发者 → 本地工具生成/修改 → 交付文件 → 闲鱼回传
```

| 角色 | 职责 | 工具 |
|------|------|------|
| 👤 小伙伴（运营） | 闲鱼商品管理、客户沟通、信息收集、交付文件、好评积累 | 闲鱼 App + 微信 |
| 👨‍💻 你（开发） | 本地 Web 工具开发维护、简历模板设计、AI 能力接入、格式导出 | 本地方案 |

**关键协作节点**：
1. 小伙伴在闲鱼接单 → 通过聊天获取用户基本信息 + 原始简历（如有）
2. 小伙伴将信息填入或转发给开发者
3. 开发者用本地工具生成/修改简历 → 导出 PDF/DOCX
4. 小伙伴将文件发给客户确认 → 修改 → 最终交付

## 三、本地交付流程

```
闲鱼客户下单
    ↓
小伙伴获取客户信息（按上表收集）
    ↓
信息转发给开发者 / 开发者自行在工具中录入
    ↓
本地工具：选择模板 → AI 优化 → 手动精调 → 导出 PDF/DOCX
    ↓
文件回传给小伙伴
    ↓
小伙伴发给客户确认 → 修改 → 最终交付 → 求好评
```

> **初期建议**：前 10 单免费或成本价做，积累真实案例截图和好评，打磨工具流程。

## 五、避坑要点

| 类别 | 说明 |
|------|------|
| 个人信息 | 客户简历含敏感信息，本地处理完毕及时删除原始文件 |
| AI 免责 | 交付时告知客户"AI 辅助生成，请自行核实内容准确性" |
| 防白嫖 | 交付前加水印预览，客户确认后再发高清版 |
| 闲鱼规则 | 简历修改归为"服务类"可发布，但如果链接外站可能被限流 |
| 沟通留痕 | 关键确认环节截图保存，避免纠纷 |
