## 一、技术栈选择

### 选型原则

- **本地运行**优先，不依赖云服务
- **Python 生态**处理文件格式（PDF/DOCX）最成熟
- **交互体验**上 React 做富文本编辑器体验远好于纯后端模板

### 技术栈表格

| 层面 | 技术 | 版本 | 说明 |
|------|------|------|------|
| **后端框架** | Python FastAPI | ≥0.110 | 异步高性能，自动生成 API 文档 |
| **ORM** | SQLAlchemy + SQLite | 2.0+ | 零配置数据库，无须安装 |
| **前端框架** | React 18 + Vite | — | 组件化开发，热更新 |
| **UI 样式** | Tailwind CSS | 3.x | 通过 CDN 或 npm 引入 |
| **状态管理** | React Context + useReducer | — | 轻量级，够用 |
| **PDF 导出** | Playwright (Python) | latest | 渲染 HTML → 高质量 PDF |
| **DOCX 导出** | python-docx | latest | 生成 .docx 文件 |
| **PDF 解析(导入)** | PyMuPDF (fitz) | latest | 从 PDF 提取文本 |
| **DOCX 解析(导入)** | python-docx | latest | 从 DOCX 提取内容 |
| **大模型接入** | httpx + DeepSeek API / MiMo API | — | 异步 HTTP 调用 |
| **模板引擎(后端)** | Jinja2 | — | 渲染简历 HTML 模板 |
| **UI 组件** | Headless UI + Heroicons | — | 无障碍、可定制 |

## 三、数据库设计

使用 SQLite，零配置，文件即数据库。

### 3.1 resumes — 简历项目表

```sql
CREATE TABLE resumes (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    title           VARCHAR(200) NOT NULL DEFAULT '未命名简历',
    customer_name   VARCHAR(100),
    customer_phone  VARCHAR(50),
    target_position VARCHAR(200),
    target_industry VARCHAR(100),
    status          VARCHAR(20) NOT NULL DEFAULT 'draft',
                    -- draft: 草稿 | processing: 处理中 | completed: 已完成
    template_id     INTEGER REFERENCES templates(id),
    raw_data        TEXT NOT NULL DEFAULT '{}',  -- JSON 存储完整简历数据
    notes           TEXT,                         -- 备注（客户要求等）
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 templates — 简历模板表

```sql
CREATE TABLE templates (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            VARCHAR(100) NOT NULL,
    description     TEXT,
    category        VARCHAR(50) NOT NULL DEFAULT 'modern',
                    -- modern: 现代 | classic: 经典 | creative: 创意 | compact: 紧凑
    thumbnail       VARCHAR(500),               -- 缩略图路径
    html_content    TEXT NOT NULL,               -- Jinja2 HTML 模板
    css_content     TEXT NOT NULL,               -- 模板 CSS
    is_default      BOOLEAN DEFAULT FALSE,
    is_active       BOOLEAN DEFAULT TRUE,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 四、后端 API 设计

### 4.1 接口概览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/resumes | 获取简历列表 |
| POST | /api/resumes | 创建新简历 |
| GET | /api/resumes/{id} | 获取单份简历详情 |
| PUT | /api/resumes/{id} | 更新简历数据 |
| DELETE | /api/resumes/{id} | 删除简历 |
| GET | /api/templates | 获取模板列表 |
| GET | /api/templates/{id} | 获取模板详情（HTML+CSS）|
| POST | /api/ai/optimize | AI 优化指定内容 |
| POST | /api/ai/rewrite | AI 改写指定内容 |
| POST | /api/ai/generate | AI 根据基本信息生成简历 |
| POST | /api/ai/jd-optimize | 根据岗位描述优化 |
| POST | /api/ai/translate | AI 中英互译 |
| POST | /api/ai/grammar | AI 语法检查 |
| POST | /api/export/pdf | 导出 PDF |
| POST | /api/export/docx | 导出 DOCX |
| POST | /api/import/parse | 解析上传的 PDF/DOCX |
| GET | /api/settings | 获取设置 |
| PUT | /api/settings | 更新设置 |

### 4.2 核心接口详细设计

#### POST /api/ai/optimize — AI 优化

```
Request:
{
  "resume_id": 1,
  "section": "work_experience",  // 要优化的板块
  "content": {                   // 该板块的原始数据
    "company": "XX公司",
    "highlights": ["负责XX项目"]
  },
  "target_position": "高级产品经理",  // 可选，目标岗位
  "target_industry": "互联网"         // 可选，目标行业
}

Response:
{
  "success": true,
  "suggestions": [
    {
      "field": "highlights",
      "original": "负责XX项目",
      "suggested": "主导XX项目全流程管理，推动项目提前两周上线，获得部门年度优秀项目奖",
      "reason": "使用量化成果+STAR法则，突出个人贡献"
    }
  ],
  "model_used": "deepseek-v3"
}
```

#### POST /api/export/pdf — 导出 PDF

```
Request:
{
  "resume_id": 1,
  "template_id": 2,
  "filename": "张三_产品经理简历.pdf"
}

Response: 直接返回 PDF 文件流（Content-Type: application/pdf）
```

#### POST /api/import/parse — 解析导入

```
Request: multipart/form-data 上传文件（.pdf / .docx）

Response:
{
  "success": true,
  "parsed_data": { ... },       // 解析后的 resume_data 结构
  "confidence": 0.85,           // 解析置信度
  "warnings": ["未能解析教育背景中的 GPA 字段"]
}
```

## 六、简历模板系统

### 6.1 模板工作原理

```
resume_data (JSON)          Jinja2 HTML 模板         最终产出
       │                          │                     │
       │     template_engine.py    │                     │
       └───────── 渲染 ────────────→   完整 HTML 页面     │
                                                        │
                          ┌──────────────────────────┐   │
                          │ Playwright → PDF          │   │
                          │ python-docx → DOCX        │   │
                          └──────────────────────────┘   │
```

### 6.2 模板示例（现代简约型 HTML 结构）

每个模板包含两个文件：`template.html` + `style.css`

```html
{# templates/modern/template.html #}
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <style>{{ css_content }}</style>
</head>
<body>
  <div class="resume-container">
    <header class="header">

### 8.1 导出 PDF（核心功能）

# services/export_service.py

async def export_pdf(resume_id: int, template_id: int) -> bytes:
    # 1. 从数据库读取简历数据
    resume = get_resume(resume_id)

    # 2. 读取模板 HTML + CSS
    template = get_template(template_id)

    # 3. 使用 Jinja2 渲染完整 HTML
    html = template_engine.render(template, resume.raw_data)

    # 4. Playwright 渲染为 PDF
    from playwright.async_api import async_playwright
    async with async_playwright() as p:
        browser = await p.chromium.launch()
        page = await browser.new_page()
        await page.set_content(html, wait_until="networkidle")
        pdf_bytes = await page.pdf(
            format="A4",
            margin={"top": "15mm", "bottom": "15mm", "left": "15mm", "right": "15mm"},
            print_background=True,
        )
        await browser.close()

    return pdf_bytes
```
